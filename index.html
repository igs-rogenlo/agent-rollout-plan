why

為什麼要為各職能建 Agent？

每個職能的工作流、工具、參考來源都不同。與其每個人自己摸索 AI，不如由部門集中建置專屬 Agent，把各職能最常用的工具與知識內建進去，讓大家上手即用。

專屬工具
每個職能內建自己需要的工具，例如：生圖工具、PSD 匯出、瀏覽器、上網查詢工具、CocosCreator 編輯、SPINE 2D 製作等等。
專屬知識
原本要給產學看的文件，也可以直接拿來當 Agent 的專屬知識庫（共用模組說明、歷史規格、競品靈感等），一份文件兩邊都能用。
集中維護
一次建好所有人受益。新版工具或文件更新，全員同步拿到。
execution

執行階段總覽

分三階段推進，先從企劃 Agent 開始落地驗證。

Phase 1 — 需求收集 本週啟動
各職能盡量把「工具需求」和「靈感來源」丟給我，形式不拘（清單、連結、截圖、口頭描述都可以）。
Phase 2 — 企劃 Agent 先行（約 3 天）
我先建出企劃 Agent，交給企劃實際試用做虎機規格。
Phase 3 — 擴展到其他職能
依各職能提供的需求，逐一建置美術/音效/機率/品檢/Client/Server Agent。
phase 1 — ask

請各職能提供兩件事

這是目前最需要大家協助的部分，決定了 Agent 能做到什麼程度。

① 工具需求

你日常工作會用到哪些工具 / 服務 / 資料來源？

  • 需要連外網、需要 VPN 的網站
  • 需要讀取的本機檔案或共用資料夾
  • 常查的文件、規格、資料庫
  • 常用的外部服務或 API
② 靈感來源

你平常從哪裡找靈感 / 參考 / 範本？

  • 競品網站、發行平台、排行榜
  • 素材庫、音效庫、參考圖庫
  • 論壇、社群、Youtube 頻道
  • 內部歷史資料、共用模組文件
產出格式不拘：條列、連結、截圖都可以。重點是越具體越好。

phase 2 — pilot

企劃 Agent 先行 — 約 3 天

取得各職能資料後，我會先把企劃 Agent 做出來，讓企劃實際拿去用。

D1 · 建置
整合企劃需要的工具（VPN 競品瀏覽、範本、共用模組文件），建立知識庫。
D2 · 調校
測試基本流程，調整 prompt 與工具串接，確保可以穩定輸出企劃產物。
D3 · 交付
交給企劃實際用來建立一支虎機規格，作為第一個落地驗證。
為什麼先做企劃？ 企劃是整個專案起點，企劃產物決定後面所有職能的工作。先讓企劃 Agent 落地，整條鏈路才有意義。
direction

方向：微創新 × 鎖定美洲

微創新
  • 基於既有成功玩法做小幅變化
  • 異動幅度小 → 風險低、排程可控
  • 更容易落地與量產
  • 產學 / 新人也能在共用模組上快速接手
鎖定美洲市場
  • 目標市場明確 → 題材、美術、數值有方向
  • 競品樣本集中 → 靈感來源好收集
  • 規格與玩法偏好可被系統化整理
  • 企劃 Agent 的知識庫可聚焦美洲偏好
核心：用 Agent 加速微創新 × 美洲市場 這條最能落地的路線。
